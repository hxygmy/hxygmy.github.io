Go 语言并发机制的初步理解

摘要：Go用channel来实现了感受上的无锁并发，实际上runtime才真正在管理并发和锁。这是一种相当优雅的封装。

1. Go语言的并发机制

Go程序中实现并发主要有两种机制

  • 通过 Go协程(Go routine) 和 信道(channel) 的协作来实现并发
  • 通过同步原语，如 sync.Mutex 等来实现并发，这是对 channel 机制的补充

协程与信道机制的设计来自一条重要的设计哲学：

“Do not communicate by sharing memory; instead, share memory by communicating.”
（不要通过共享内存来通信，通过通信来共享内存。）—Effective Go
